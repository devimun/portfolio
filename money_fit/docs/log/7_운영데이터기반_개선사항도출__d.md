# 운영 데이터 기반 개선사항 도출

## 분석 개요

2025-09-25 ~ 2025-10-22 약 4주 기간 동안, 사용자들이 앱 설치 후 핵심 기능까지의 이탈 지점을 확인하고 개선 우선 순위를 정의합니다.
데이터 소스는 앱 내 이벤트들을 활용해 Analytics 데이터를 통해 획득하였습니다.

## 전환 성과 요약

단계 명칭

- A : 앱 설치 후 실행
- B : 실행 -> 듀토리얼 완료
- C : 듀토리얼 완료 -> 지출 등록 (핵심 경험)

| 단계 | 9/25~9/27 | 9/28~10/4 | 10/5~10/11 | 10/12~10/18 | 10/19~10/22 | 누적 |
| ---- | --------- | --------- | ---------- | ----------- | ----------- | ---- |
| A    | 12        | 26        | 34         | 20          | 6           | 98   |
| B    | 4         | 3         | 14         | 6           | 2           | 29   |
| C    | 2         | 1         | 4          | 2           | 1           | 10   |

데이터 결과를 분석하면서, 몇 가지 사안을 도출할 수 있었습니다.

1. 약 70%의 사용자들이 앱을 설치하고 듀토리얼도 완료하지 않은 채로 앱을 삭제하고 있다는 점
2. 하지만 듀토리얼을 완료한 경우 앱의 핵심 경험까지 연결되는 사용자가 34%로 낮지 않다는 점이었습니다.

이를 통해 설치 직후 이탈률을 낮추기 위한 방법을 찾고자 했으며, 플레이스토어 상위 2개의 가계부 앱을 확인하고 개선점을 찾고자 했습니다.

## 개선 요소 진단

상위 2개의 가계부 앱과 비교해 봤을 때 크게 2가지 요소에서 큰 차이점이 있다고 판단했습니다.

| 요소         | 머니핏                    | 상위 가계부앱             |
| ------------ | ------------------------- | ------------------------- |
| 앱 첫 실행   | 듀토리얼 / 오픈 앱 광고   | 메인 화면                 |
| 커스터마이징 | 테마 고정, 다크 모드 지원 | 테마 변경, 다크 모드 지원 |

1. 상위 가계부 앱의 경우 앱이 실행됐을 때 즉시 앱의 메인 화면을 보여주어 사용자들의 몰입도를 해치지 않을 수 있었습니다. 하지만 머니핏의 경우 '오픈 앱 광고'와 '듀토리얼'과 같은 요소들로 인해 흐름이 단절되어 몰입도를 해칠 수 있다고 판단했습니다.
2. 또한 머니핏의 경우 고정된 테마와 다크 모드만을 지원하여 사용자들의 커스터마이징 요소가 제한적이었지만, 상위 앱의 경우 테마 커스터마이징 요소가 존재하여, 보다 다양한 사용자들을의 UI적 요구를 적용할 수 있었습니다.

## 개선 방향

1. **핵심 경험 즉시 제공**
   - 앱의 핵심 가치와 목표를 전달하는 것도 중요하지만, 가계부의 경우엔 대부분의 사용자가 앱의 목적과 사용법에 대해서 인지하고 있을 것이라고 가정하고, 듀토리얼을 제거하고 즉시 핵심 경험을 제공할 수 있도록 변경합니다.
2. **UX 흐름 단절 최소화**
   - 앱 실행 시 노출되던 오픈 앱 광고를 비활성화 합니다.
3. **UI 커스터마이징 제공**
   - 메인 색상 팔레트와 다크 모드를 사용자 설정으로 분리합니다.
   - 디자인 실험을 위해 별도의 색상 샌드박스 프로젝트를 선행한 뒤 본 앱에 적용합니다.

## 향후 계획

- 개선안을 반영한 업데이트 후 마케팅을 통해, 신규 사용자를 유입하고 다시 데이터를 측정합니다.
